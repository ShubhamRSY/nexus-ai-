What Is Nexus?

Nexus is an open-source, omnichannel AI agent platform built for customer support and contact centres. It replaces three separate tools — live chat, AI copilot, and phone systems — with one AI-powered console. A single orchestrator routes conversations across channels, remembers context, retrieves knowledge, and streams responses in real time.

Core Components

API Gateway (FastAPI)

Single Python application serving the web console, REST API, and WebSocket connections. Middleware stack handles CORS, tenant isolation, rate limiting, and authentication.

Agent Orchestrator

Routes inbound requests to the correct channel handler, builds channel-specific prompts, manages conversation state, and executes tool calls (RAG retrieval, CRM lookups) before returning the LLM response.

LLM Client

Abstract client layer supporting OpenAI GPT-4o, Anthropic Claude 3.5, and Google Gemini 2.0. Configurable per conversation.

RAG Pipeline

Retrieval-augmented generation with vector database (FAISS/chroma). Sources include citations and grounding metrics on every response.

Sourcesrc/rag/

Feedback Engine

Post-interaction CSAT surveys feed an auto-adjustment engine. Scores trigger automatic tuning of agent personality, response length, escalation thresholds, and tone.

Sourcesrc/feedback/engine.py
API/api/v1/feedback/ — report, analyze, snapshot, config, suggestions, auto-adjust

Integrations Vault

Encrypted store for all third-party API keys (AES-256-GCM). Never logged, never exposed, never hardcoded.

Sourcesrc/integrations/vault.py

Database

SQLite with schema migrations. Tables for sessions, messages, tool calls, feedback loop config, performance trends, and encrypted credentials.

Sourcesrc/database.py

Channels Detail

Chat

Full-featured web chat interface with real-time streaming via Server-Sent Events. Multi-line input, conversation history, tool call visibility.

Copilot

Agent-assist mode. Paste a ticket transcript, Nexus analyses it and generates a suggested reply. Human reviews before sending — keeps the agent in the loop.

Voice (Twilio)

Inbound and outbound PSTN calls. Live STT transcription streams to the console. AI responses spoken via TTS (OpenAI or ElevenLabs). Calls recorded and logged.

Setup guidedocs/integrations/twilio-setup.md

Voice (Amazon Connect)

Lambda-style webhook handler. Accepts JSON with contact flow attributes, processes through the orchestrator, returns JSON for branch conditions.

Voice (Generic CCaaS)

Abstract CcaasVoiceHandler base class. Implement handle_inbound, handle_process, handle_status_callback to add any SIP or CCaaS provider.

Security

  • Credentials: AES-256-GCM encrypted at rest in the Integrations Vault. Never logged, never exposed in API responses.
  • Authentication: JWT-based with configurable expiry. Tenant isolation at middleware level.
  • Input validation: Pydantic models on all API endpoints. SQL injection prevented via parameterised queries.
  • CORS: Strict origin whitelist. No wildcard in production.

Quick Start

git clone https://github.com/ShubhamRSY/voice-agents.git
cd voice-agents
pip install -e ".[dev]"
cp .env.example .env    # add your API keys
uvicorn src.main:app --reload --port 8001

Open http://localhost:8001 in a browser.

Testing

# Unit tests
pytest tests/ --timeout=60 -v

# E2E tests (requires running server on port 8001)
pytest tests/test_comprehensive_e2e.py --timeout=120 -v

# Lint + type check
ruff check src/ scripts/
mypy src/ --ignore-missing-imports
